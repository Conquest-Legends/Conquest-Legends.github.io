Conquest_Legends' Code Style for OI

2021-06-22 通用

本篇博文介绍下自己目前的代码风格。

当然不是说自己的风格多好，普及组选手为什么要说自己的代码风格多么多么好……只是提供一种良好的代码风格。

当然，不要照抄，照抄的话考试时反作弊system会把我拉下水的。

什么是代码风格

代码风格，就是你个人在打代码的时候的习惯。

每个人的风格都不一样，因人而异。

代码风格对于你程序的查错，书写，可读性起着很大的作用。

为什么要养成良好的代码风格

很简单，看着舒服，debug更加容易，打起来也有一种感觉自己是dalao的感觉。

良好的代码风格也许会让你付出1.5倍打代码的时间，但是会为你节省至少2倍的Debug的时间。

本人的代码风格

概览：

  1. 所有头文件，必须使用c前缀，不用.h后缀，用什么头文件就写什么头文件，不能滥写头文件，不应使用万能头文件。

  2. 应当使用 using namespace std，普及组选手写上就行。

  3. 输入，初始化，功能全部用特定的函数使用，并在主函数中按顺序调用，节省运行空间，比如输入时自己定义一个函数名为 inp 之类的。

  4. 写代码时，应带上 freopen 并予以注释。

  5. main函数置于末尾，并在括号里边写一个 void。

  6. 判断时，不应用int类型代替bool。

  7. 使用and or 代替 && ||。

缩进

  1. 每个代码块，予以Tab或4个空格的缩进。

花括号

  1. 无论是什么语句，if,for还是别的什么也罢，必须加上花括号。

  2. 花括号必须严格按照【花括号不换行】，又名【后花括号】。

这么写:

行

  1. 除for循环以外，每行只有一条语句。

  2. 多个意义独立的代码块之间，用换行隔开，例如输入与输出。

  3. using namespace std;必须紧跟在头文件之后，并且与头文件必须有一个换行。

  4. 成员函数，函数，结构体，全局变量块必须用换行隔开。

变量

  1. 变量命名时，采用驼峰命名法或蛇形命名法。

比如：

int ThisIsTheStrongestDxr 驼峰命名法

int this_is_the_strongest_dxr 蛇形命名法

我个人比较喜欢蛇形，很少写驼峰。

  1. 在定义结构体时，首字母必须大写，其他字母除驼峰的需要外小写，驼峰 $or$ 蛇形。

如：

struct node 错误样例

struct DxrNb 正确样例

struct Dxr_tql 又是一个正确样例

空格

  1. 像> < = == >= <= !=这些判断用的，两边的变量或常量必须加上空格。

  2. 逗号,与for中的分号;，后面必须有一个空格，前面不能有空格。

  3. 三目运算符两侧，冒号:两侧必须有空格。

  4. #include 与后面的文件名称必须有一个空格。

  5. 对于注释，必须在同一缩进的基础上与前面的语句有一个空格，并且与后面注释的内容有一个空格。

如：

#include <cstdio>
